# Paper Outline

Title: **A Multidimensional Ontology of Unix/Linux Terminal Emulators**

## 1. Abstract
Introduce the terminal emulator ecosystem, the need for a multidimensional ontology, and the hybrid public/scientific design.

## 2. Introduction
Terminals as research/workbench infrastructure, developer culture, Unix/Linux heritage, and open-source software ecosystems.

## 3. Scope and Inclusion Criteria
Define terminal emulator; exclude multiplexers, shells, embedded IDE terminals, and wrappers.

## 4. Hybrid Ontological Design
State explicitly that the ontology has a formal reproducible layer and an interpretive public-facing layer.

## 5. Dimensions and Data Model
Describe technical, interaction, temporal, lineage, provenance, and derived dimensions.

## 6. Validation and Curation
Describe pruning, source tracking, confidence levels, scope checks, and schema validation.

## 7. Periodic Table Mapping
Explain how the visual layout translates multidimensional structure into an exploratory public interface.

## 8. Discussion
Interpret what the ontology reveals about terminal emulator evolution.

## 9. Limitations
Acknowledge incomplete metadata, lineage uncertainty, and historical source limitations.

## 10. Data and Software Availability
Point to GitHub, Zenodo DOI, dataset exports, and license.
