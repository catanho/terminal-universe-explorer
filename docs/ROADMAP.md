# Roadmap

## v2 (current)
Explorer-first interface with curated dataset.

Focus:
- filtering and grouping
- clean interaction model
- lightweight, static architecture

---

## v3 — Scientific Layer

Introduce a deeper analytical view of the dataset.

### Goals

- expose full ontology fields
- show lineage relationships explicitly
- support more rigorous comparisons

---

## Planned features

### 1. Scientific Mode (real, not placeholder)

- toggle between Explorer and Scientific views
- expose:
  - lineage concepts
  - confidence levels
  - source-backed attributes

---

### 2. Lineage visualization

- graph view of terminal evolution
- parent/child relationships
- conceptual inheritance

---

### 3. Data transparency

- per-field provenance visibility
- confidence indicators
- source links per attribute

---

### 4. Advanced sorting and metrics

- activity signal refinement
- maturity scoring improvements
- possible composite metrics

---

### 5. Dataset expansion

- fill missing historical data (e.g. release years)
- refine language classification
- expand coverage where meaningful

---

## Longer term

- API or export layer (optional)
- integration with external datasets
- community-driven curation

---

## Philosophy

v3 should not make the tool heavier.

It should make the structure clearer.
