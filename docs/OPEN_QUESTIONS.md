# Open Questions

This project is a working model of the terminal emulator design space.

Some areas are intentionally unresolved.

---

## Classification

- Are the current dimensions (engine, rendering, surface, ecosystem, status) sufficient?
- Are there better ways to group terminal architectures?

---

## Engine families

- How should “engine family” be defined more rigorously?
- Where do hybrid or modern GPU terminals fit?

---

## Language model

- Is “primary language” the right abstraction?
- How should multi-language implementations be represented?

---

## Lineage

- What is the best way to represent inheritance and influence?
- Should lineage be strict (parent-child) or conceptual?

---

## Missing data

- Many entries lack reliable first release years
- Some classifications rely on interpretation rather than explicit documentation

---

## Scope

- What should be included or excluded?
- Where is the boundary between terminal emulator and terminal-like interface?

---

## Interface

- Does the current Explorer view make structure visible?
- What would make relationships clearer?

---

## Philosophy

This project is not trying to be final.

It is trying to be useful.
